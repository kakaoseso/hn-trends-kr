오디오 장비 시장은 엔지니어의 관점에서 볼 때 참으로 기묘한 곳이다. 아주 기초적인 디지털 신호 처리(DSP) 기술이 들어간 장비조차 '하이파이'라는 수식어가 붙으면 수백, 수천 달러의 가격표를 달고 나온다. MiniDSP 같은 장비가 훌륭한 솔루션이긴 하지만, 단순히 방 안의 룸 코렉션(Room Correction)이나 액티브 크로스오버를 구성하기 위해 매번 200달러 이상을 지출하는 것은 가끔 억울하게 느껴진다.

최근 Hacker News를 뜨겁게 달군 **DSPi** 프로젝트는 이러한 불만에 대한 완벽한 기술적 해답을 제시한다. 단돈 4달러짜리 Raspberry Pi Pico(RP2040/RP2350)를 풀스택 USB 디지털 오디오 프로세서로 탈바꿈시키는 이 펌웨어는, 단순한 취미용 장난감이 아니라 프로덕션 레벨의 엔지니어링이 돋보이는 작품이다.

솔직히 처음엔 그저 그런 장난감 프로젝트인 줄 알았다. 하지만 내부 아키텍처와 코드를 뜯어보고 나니, 이 프로젝트가 마이크로컨트롤러의 한계를 어떻게 영리하게 우회했는지 감탄할 수밖에 없었다. 15년 넘게 임베디드와 백엔드 시스템을 다뤄온 입장에서, DSPi가 왜 기술적으로 훌륭한지 깊이 파헤쳐보자.

## 클럭 스위칭 글리치를 원천 차단한 시스템 설계

오디오 프로그래밍을 해본 사람이라면 샘플 레이트(44.1, 48, 96kHz)가 바뀔 때마다 발생하는 클럭 동기화 문제와 팝핑(Popping) 노이즈가 얼마나 골치 아픈지 알 것이다. DSPi는 이 문제를 아주 무식하지만 가장 확실한 방법으로 해결했다.

시스템 클럭을 **307.2 MHz** 로 고정해버린 것이다.

RP2040의 기본 클럭은 133MHz지만, 약간의 코어 전압(1.15V) 상승을 통해 307.2MHz로 오버클럭했다. 왜 하필 307.2MHz일까? 이 주파수는 오디오에서 주로 사용하는 모든 샘플 레이트의 정수배를 만들어낼 수 있는 매직 넘버다. 덕분에 샘플 레이트가 변경될 때 시스템 클럭을 재조정(retune)할 필요 없이 PIO 디바이더만 정수로 변경하면 된다. 클럭 스위칭으로 인한 글리치가 수학적으로 발생할 수 없는 구조를 만든 것이다.

## Biquad와 SVF의 하이브리드: 수학적 한계를 극복하다

이 프로젝트에서 가장 인상 깊었던 부분은 RP2350의 하드웨어 FPU를 활용한 **EQ 필터 아키텍처** 다. 

일반적으로 가벼운 DSP 엔진들은 모든 대역에 전통적인 Biquad(Transposed Direct Form II) 필터를 사용한다. 하지만 Biquad 필터는 치명적인 단점이 있다. 샘플 레이트 대비 매우 낮은 저주파(예: 서브우퍼 대역)를 타겟으로 할 때, 필터 계수(Coefficient)의 양자화 에러에 극도로 취약해진다는 점이다. 이는 저음역대에서 의도치 않은 노이즈나 부정확한 커브를 만들어낸다.

DSPi는 이를 해결하기 위해 하이브리드 접근법을 택했다.
- **고주파 대역:** 전통적인 TDF2 Biquad 필터 사용 (연산량 최소화)
- **저주파 대역 (Fs/7.5 이하):** Cytomic SVF (State Variable Filter) 토폴로지 적용

SVF는 Biquad보다 연산량이 많지만 저주파 대역에서 수치적 정확도가 압도적으로 높다. RP2350의 싱글 프리시전(Single-Precision) Float 파이프라인과 하드웨어 FPU를 믿고, 주파수 대역에 따라 필터 토폴로지를 동적으로 전환하는 설계는 오디오 DSP의 맹점을 정확히 꿰뚫어 본 엔지니어의 솜씨다.

## PIO와 듀얼 코어의 극한 활용

Raspberry Pi Pico의 꽃은 단연코 PIO(Programmable I/O)다. DSPi는 CPU가 해야 할 지루한 I/O 작업을 PIO에 전면 오프로딩했다.

S/PDIF 인코딩(BMC/NRZI)과 서브우퍼용 PDM 비트스트림 생성은 모두 PIO가 백그라운드에서 처리한다. 덕분에 CPU는 오로지 수학적 연산(EQ, 매트릭스 믹싱, 크로스피드 등)에만 집중할 수 있다.

특히 **PDM 서브우퍼 출력** 은 천재적인 해킹이다. Core 1을 통째로 2nd-order Delta-Sigma 모듈레이터 전용으로 할당하여, 별도의 외부 DAC 없이도 저항과 커패시터(Low-pass filter)만으로 서브우퍼 앰프에 직결할 수 있는 아날로그 신호를 만들어낸다. 256x 오버샘플링과 TPDF 디더링까지 적용되어 있어, 서브우퍼 용도로는 상용 장비와 구분할 수 없는 수준의 퀄리티를 낸다.

```c
// 매트릭스 라우팅 패킷 구조를 보면 이 펌웨어가 얼마나 유연한지 알 수 있다.
struct __attribute__((packed)) {
    uint8_t input;        // 0-1 (USB L/R)
    uint8_t output;       // RP2350: 0-8, RP2040: 0-4
    uint8_t enabled;      // 0 or 1
    uint8_t phase_invert; // 0 or 1
    float gain_db;        // -inf to +12dB
}
```

## Hacker News 커뮤니티의 반응과 Latency 논쟁

HN 스레드에서는 이 프로젝트를 CamillaDSP나 Dirac Live와 비교하는 흥미로운 토론이 이어졌다. 

한 유저는 라즈베리파이 3에 Linux와 CamillaDSP를 올려 룸 코렉션을 구현한 경험을 공유했다. 훌륭한 접근이지만, Linux 기반 시스템은 부팅 시간, SD 카드 커럽션, OS 레벨의 오버헤드라는 단점이 있다. 반면 DSPi는 RTOS조차 없는 Bare-metal 펌웨어다. 전원을 넣자마자 즉시 켜지며, USB 오디오 인터페이스로 네이티브하게 작동한다.

하지만 비판적인 시각도 존재했다. 바로 **레이턴시(Latency)** 문제다.
저자가 밝힌 DSPi의 nominal latency는 약 8ms(±1ms)다. 홈 오디오 감상이나 다채널 액티브 크로스오버 용도로는 완벽한 수치다. 스피커 유닛 간의 위상(Phase)만 맞으면 되기 때문이다.

그러나 라이브 악기 연주자나 실시간 모니터링이 필요한 팟캐스터에게 8ms는 꽤 거슬리는 수치다. 소리가 공기 중에서 약 2.7미터를 날아오는 것과 같은 지연이기 때문이다. 만약 당신이 이 장비를 기타 이펙터 프로세서나 신디사이저의 실시간 DSP로 쓰려 했다면, 다른 대안(예: Teensy 4.0)을 찾는 것이 정신 건강에 좋을 것이다.

## 결론: 그래서 쓸만한가?

이 프로젝트는 마이크로컨트롤러 생태계가 얼마나 성숙했는지 보여주는 완벽한 사례다. 과거라면 전용 DSP 칩(SHARC 등)이나 무거운 FPGA가 필요했던 작업들을, 이제는 4달러짜리 범용 칩셋과 최적화된 C/ARM 어셈블리 코드로 해결할 수 있게 되었다.

만약 당신이 3-way 액티브 스피커를 직접 설계하고 싶거나, 방 안의 부밍(Booming)을 잡기 위해 파라메트릭 EQ가 필요하다면, 비싼 상용 장비를 사기 전에 DSPi를 먼저 시도해보라. 남는 핀을 활용해 I2S DAC 몇 개만 연결하면, 수백 달러짜리 하이엔드 장비가 전혀 부럽지 않은 나만의 완벽한 오디오 파이프라인을 구축할 수 있을 것이다.
