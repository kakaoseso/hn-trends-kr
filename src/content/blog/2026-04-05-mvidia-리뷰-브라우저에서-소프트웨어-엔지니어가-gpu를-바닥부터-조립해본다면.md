우리 소프트웨어 엔지니어들은 종종 거대한 추상화의 탑 꼭대기에서 살아갑니다. PyTorch로 모델을 학습시키거나 CUDA 커널을 최적화할 때도, 그 밑단에서 수십억 개의 트랜지스터가 어떻게 스위칭되는지 정확히 아는 사람은 드물죠. 저 역시 15년 넘게 백엔드와 분산 시스템을 다루면서 하드웨어는 그저 '빠르게 동작해야 하는 블랙박스' 정도로 취급하곤 했습니다.

그런데 최근 Hacker News에서 꽤 흥미로운 프로젝트를 하나 발견했습니다. 이름부터 심상치 않은 Mvidia입니다. 브라우저 위에서 NMOS와 PMOS 트랜지스터부터 시작해 논리 게이트를 만들고, 궁극적으로는 GPU 아키텍처를 조립해 나가는 퍼즐 게임입니다.

## 트랜지스터부터 시작하는 Bottom-Up 접근

이 게임은 단순히 AND, OR 게이트를 던져주고 시작하는 Nand2Tetris 류의 프로젝트와는 궤를 달리합니다. 첫 레벨부터 트랜지스터의 물리적 특성을 이해하고 배선해야 합니다.

가장 인상 깊었던 부분은 Tri-state logic(0, 1, High-Z)을 시뮬레이션 엔진에 구현했다는 점입니다. 소프트웨어 엔지니어들에게는 0과 1의 세계가 익숙하지만, 실제 하드웨어에서는 연결되지 않은 상태(High Impedance, Z)를 다루는 것이 필수적입니다. 이 게임은 NMOS를 이용해 Pull-down 네트워크를 구성하고, 입력이 없을 때 Z 상태가 되는 것을 직접 눈으로 확인하게 만듭니다.

## 추상화 누수(Abstraction Leak)와 시뮬레이션의 한계

하지만 시니어 엔지니어의 시각에서 볼 때, 이 프로젝트의 시뮬레이션 엔진은 몇 가지 뚜렷한 한계를 노출합니다. 가장 대표적인 것이 레벨 2.7의 1T1C(1 Transistor 1 Capacitor) DRAM 셀 구현입니다.

실제 아날로그 회로에서 커패시터는 전하를 저장하는 물리적 소자일 뿐, 별도의 Enable 핀 같은 것은 존재하지 않습니다. 하지만 Mvidia의 기반 엔진은 철저히 디지털 상태 머신(Digital State Machine)으로 설계된 탓에, 커패시터를 순수한 아날로그 소자로 시뮬레이션하지 못했습니다. 결국 개발자는 커패시터 블록에 인위적인 Enable 게이트를 추가하는 타협을 해야만 했죠.

이건 우리가 소프트웨어 아키텍처를 설계할 때 흔히 겪는 Abstraction leak의 전형적인 사례입니다. 특정 도메인(여기서는 디지털 논리)에 최적화된 코어 엔진을 가지고 다른 도메인(아날로그 특성)을 억지로 표현하려다 보니 인터페이스가 어색해진 것입니다.

## LLM 기반 개발과 UX의 함정

개발자는 이 게임의 훌륭한 UI와 복잡한 와이어 라우팅 알고리즘을 구현하는 데 Claude의 도움을 크게 받았다고 밝혔습니다. 2024년의 사이드 프로젝트 워크플로우로서는 매우 훌륭한 접근입니다. 사실 저도 최근 사내 툴링을 만들 때 비슷한 방식으로 LLM을 적극 활용하고 있으니까요.

그러나 도메인 지식이 결여된 UX 디자인이 얼마나 치명적인지 보여주는 사례도 있었습니다. HN 댓글란을 보면 10년 이상 Mixed-signal IC를 설계한 베테랑 엔지니어가 첫 번째 레벨을 통과하지 못해 좌절한 에피소드가 나옵니다. 원인은 황당하게도 배경 그리드의 굵은 회색 선이 마치 기본적으로 연결된 '와이어'처럼 보였기 때문입니다. Cadence Virtuoso 같은 실제 회로 설계 툴들이 왜 실선 대신 점(Dot) 그리드를 사용하는지 알 수 있는 대목입니다.

또한, 게이미피케이션(Gamification) 요소로 넣은 타이머 기반 미니게임들은 엔지니어링 학습 도구로서는 최악의 선택이었습니다. DRAM Refresh를 설명한답시고 유저에게 1분 동안 8개의 로우를 계속 클릭하게 만드는 미니게임은 "10초면 원리를 이해하는데 왜 1분을 낭비하게 만드냐"는 원성을 샀습니다. 학습 도구에서의 반복 작업은 피로감만 가중시킬 뿐입니다.

## Hacker News 커뮤니티의 반응

HN 커뮤니티의 반응은 전반적으로 열광적이면서도 날카로웠습니다. 사람들은 이 프로젝트를 보며 다음과 같은 명작들을 떠올렸습니다.

- **Turing Complete:** 스팀에 있는 훌륭한 CPU 빌딩 게임입니다.
- **Silicon Zeroes:** 이진수의 복잡성을 덜어내고 회로 설계에 집중하게 해주는 게임입니다.
- **Zachtronics 시리즈:** KOHCTPYKTOP 같은 하드코어 엔지니어링 퍼즐 게임들 말이죠.

특히, 비전공자나 하드웨어 지식이 없는 소프트웨어 개발자들을 위해 초반 튜토리얼(Act 0)이 반드시 필요하다는 피드백이 많았습니다. 개발자도 이를 수용하여 다음 업데이트에 반영하기로 한 점은 오픈소스 생태계의 긍정적인 피드백 루프를 잘 보여줍니다.

## 총평: 장난감 이상, 프로덕션 미만

Mvidia는 아직 거칠고 다듬어지지 않은 다이아몬드 원석 같습니다. 시뮬레이션 엔진의 엣지 케이스 처리나 UX 측면에서는 아쉬운 점이 많지만, 브라우저 환경에서 이 정도 수준의 하드웨어 교육용 툴을 혼자(그리고 LLM과 함께) 만들어냈다는 점은 박수받아 마땅합니다.

단순히 API 문서를 읽고 CRUD를 찍어내는 것에 지쳤다면, 이번 주말에는 Mvidia를 켜고 NAND 게이트부터 직접 조립하며 우리가 매일 혹사시키는 GPU의 밑바닥을 경험해 보시길 권합니다.
